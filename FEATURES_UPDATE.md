# 功能更新说明

## 🎉 新增功能

### 1. 失物招领与失物找寻分离

系统现在支持两种类型的信息发布：

#### 🔍 失物招领（Found）
- 用于发布捡到的物品信息
- 帮助失主找回丢失的物品
- 显示"发现地点"

#### 📢 失物找寻（Lost）
- 用于发布丢失物品的找寻信息
- 帮助寻找丢失的物品
- 显示"丢失地点"

### 2. 地理位置定位功能

#### 📍 自动定位
- 点击"定位"按钮自动获取当前位置
- 使用浏览器地理定位API
- 支持高精度定位

#### 🗺️ 地址解析
- 自动将经纬度转换为可读地址
- 使用OpenStreetMap的Nominatim API
- 支持中文地址显示

#### 🌐 地图查看
- 每条信息都显示"查看地图"链接
- 点击后跳转到Google Maps
- 显示精确的地理位置

## 📋 使用方法

### 发布失物招领信息

1. 点击"🔍 失物招领"标签
2. 填写物品信息：
   - 物品名称
   - 物品类别
   - 发现地点（可点击"定位"按钮自动获取）
   - 联系方式
   - 详细描述
3. 点击"发布招领信息"

### 发布失物找寻信息

1. 点击"📢 失物找寻"标签
2. 填写物品信息：
   - 物品名称
   - 物品类别
   - 丢失地点（可点击"定位"按钮自动获取）
   - 联系方式
   - 详细描述
3. 点击"发布找寻信息"

### 查看信息列表

1. 点击"🔍 失物招领"查看招领信息
2. 点击"📢 失物找寻"查看找寻信息
3. 使用搜索框搜索物品
4. 使用类别筛选器过滤信息

### 使用定位功能

1. 在发布表单中找到地点输入框
2. 点击"📍 定位"按钮
3. 浏览器会请求位置权限
4. 允许后自动获取当前位置
5. 系统会自动填充地址信息

## 🔧 技术实现

### 前端技术
- **地理定位API**: `navigator.geolocation`
- **逆地理编码**: OpenStreetMap Nominatim API
- **地图链接**: Google Maps

### 后端技术
- **数据库字段**:
  - `type`: 信息类型（found/lost）
  - `latitude`: 纬度
  - `longitude`: 经度

### 数据库迁移
- 自动迁移脚本：`migrate_database.py`
- 保留原有数据
- 添加新字段并设置默认值

## ⚠️ 注意事项

### 地理定位权限
- 首次使用定位功能需要授权
- 部分浏览器可能需要HTTPS
- 移动端定位更准确

### 地址解析
- 依赖OpenStreetMap API
- 可能有请求频率限制
- 部分地区地址可能不准确

### 数据库迁移
- 迁移前建议备份数据库
- 迁移后无法回退
- 确保应用已停止运行


## 📊 数据结构

### LostItem 表结构

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer | 主键 |
| type | String(20) | 类型：found/lost |
| title | String(100) | 物品名称 |
| description | Text | 详细描述 |
| category | String(50) | 物品类别 |
| location | String(100) | 地点描述 |
| latitude | Float | 纬度 |
| longitude | Float | 经度 |
| contact | String(50) | 联系方式 |
| status | String(20) | 状态 |
| user_id | Integer | 用户ID |
| created_at | DateTime | 创建时间 |
| updated_at | DateTime | 更新时间 |
