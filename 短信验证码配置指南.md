- ✅ 验证码功能已正常工作

## 短信服务集成方案

### 方案一：阿里云短信服务

#### 1. 注册阿里云账号
1. 访问 [阿里云官网](https://www.aliyun.com/)
2. 注册并完成实名认证

#### 2. 开通短信服务
1. 进入 [云通信控制台](https://dysms.console.aliyun.com/)
2. 点击"添加签名"并提交审核
   - 签名名称：如"校园失物招领"
   - 签名来源：选择"验证码"或"通用"
3. 点击"添加模板"并提交审核
   - 模板内容：`您的验证码是${code}`
   - 模板类型：验证码

#### 3. 获取AccessKey
1. 进入 [AccessKey管理](https://ram.console.aliyun.com/manage/ak)
2. 创建AccessKey
3. 记录 **AccessKey ID** 和 **AccessKey Secret**

#### 4. 安装SDK
```bash
pip install alibabacloud_dysms20170525
```

#### 5. 配置环境变量
在 `.env` 文件中添加：
```env
# 阿里云短信配置
ALIYUN_ACCESS_KEY_ID=your_access_key_id
ALIYUN_ACCESS_KEY_SECRET=your_access_key_secret
ALIYUN_SIGN_NAME=校园失物招领
ALIYUN_TEMPLATE_CODE=SMS_xxxxx
```

#### 6. 修改后端代码

在 `app.py` 中添加阿里云短信发送函数：

```python
from alibabacloud_dysms20170525.client import DysmsClient as Dysms
from alibabacloud_tea_openapi import models as dysms_models
import json
import os

def send_sms_aliyun(phone, code):
    """使用阿里云发送短信"""
    # 创建客户端
    client = Dysms(
        access_key_id=os.getenv('ALIYUN_ACCESS_KEY_ID'),
        access_key_secret=os.getenv('ALIYUN_ACCESS_KEY_SECRET'),
        endpoint='dysms.aliyuncs.com'
    )
    
    # 构建请求
    send_sms_request = dysms_models.SendSmsRequest(
        sign_name=os.getenv('ALIYUN_SIGN_NAME'),
        template_code=os.getenv('ALIYUN_TEMPLATE_CODE'),
        phone_numbers=phone,
        template_param=f'{{"code":"{code}"}}'
    )
    
    # 发送短信
    try:
        response = client.send_sms(send_sms_request)
        print(f'[短信] 发送成功 - 手机号: {phone}, 验证码: {code}')
        return True
    except Exception as e:
        print(f'[短信] 发送失败: {e}')
        return False
```

### 方案二：腾讯云短信服务

#### 1. 注册腾讯云账号
1. 访问 [腾讯云官网](https://cloud.tencent.com/)
2. 注册并完成实名认证

#### 2. 开通短信服务
1. 进入 [短信控制台](https://console.cloud.tencent.com/sms)
2. 创建短信签名
   - 签名内容：如"校园失物招领"
3. 创建短信正文模板
   - 模板内容：`您的验证码是{1}`
   - 模板类型：验证码

#### 3. 获取密钥
1. 进入 [API密钥管理](https://console.cloud.tencent.com/cam/capi)
2. 创建密钥
3. 记录 **SecretId** 和 **SecretKey**

#### 4. 安装SDK
```bash
pip install tencentcloud-sdk-python
```

#### 5. 配置环境变量
在 `.env` 文件中添加：
```env
# 腾讯云短信配置
TENCENT_SECRET_ID=your_secret_id
TENCENT_SECRET_KEY=your_secret_key
TENCENT_APP_ID=your_app_id
TENCENT_SIGN_NAME=校园失物招领
TENCENT_TEMPLATE_ID=xxxxxx
```

#### 6. 修改后端代码

在 `app.py` 中添加腾讯云短信发送函数：

```python
from tencentcloud.common import credential
from tencentcloud.common.profile.client_profile import ClientProfile
from tencentcloud.sms.v20210111 import sms_client
from tencentcloud.sms.v20210111 import models
import os

def send_sms_tencent(phone, code):
    """使用腾讯云发送短信"""
    # 初始化认证信息
    cred = credential.Credential(
        os.getenv('TENCENT_SECRET_ID'),
        os.getenv('TENCENT_SECRET_KEY')
    )
    
    # 创建客户端
    client = sms_client.SmsClient(cred, "ap-guangzhou")
    
    # 构建请求
    req = models.SendSmsRequest()
    req.PhoneNumberSet = [f"+86{phone}"]
    req.TemplateId = os.getenv('TENCENT_TEMPLATE_ID')
    req.TemplateParamSet = [code]
    req.SmsSdkAppId = os.getenv('TENCENT_APP_ID')
    req.SignName = os.getenv('TENCENT_SIGN_NAME')
    
    # 发送短信
    try:
        resp = client.SendSms(req)
        print(f'[短信] 发送成功 - 手机号: {phone}, 验证码: {code}')
        return True
    except Exception as e:
        print(f'[短信] 发送失败: {e}')
        return False
```

## 安全建议

1. **验证码有效期**
   - 当前设置：5分钟
   - 建议范围：3-10分钟

2. **验证码长度**
   - 当前设置：6位数字
   - 建议不要少于6位

3. **发送频率限制**
   - 建议添加：同一手机号60秒内只能发送一次
   - 建议添加：同一手机号每天最多发送10次

4. **日志记录**
   - 记录每次发送的手机号、验证码、时间
   - 用于审计和问题排查

## 常见问题

### Q: 点击"获取验证码"没有反应？
A: 检查以下几点：
1. 按F12打开浏览器控制台，查看是否有错误
2. 检查网络连接
3. 检查服务器是否正常运行
4. 检查手机号格式是否正确（11位数字）

### Q: 验证码显示为 undefined？
A: 检查后端返回的数据格式，确保返回了 `code` 字段

### Q: 提示"验证码不存在或已过期"？
A: 可能原因：
1. 验证码已超过5分钟有效期
2. 服务器重启导致验证码缓存清空
3. 输入了错误的手机号

### Q: 生产环境如何禁用显示验证码？
A: 修改前端代码，移除显示验证码的提示：

**login.html 第174-179行：**
```javascript
if (response.ok) {
    alert('验证码已发送到您的手机');
    // 移除验证码显示逻辑
    // if (data.code) { ... }
}
```

**script.js 第685-687行：**
```javascript
if (response.ok) {
    alert('验证码已发送到您的手机');
    // 移除验证码显示逻辑
    // `验证码：${data.code}（实际应用中验证码将通过短信发送）`
}
```

### 验证流程
```
用户点击"获取验证码"
    ↓
生成6位随机验证码
    ↓
存储到字典（5分钟过期）
    ↓
调用短信API发送（开发环境：打印到控制台）
    ↓
前端接收响应并显示验证码
    ↓
用户输入验证码提交
    ↓
后端验证：手机号 + 验证码 + 未过期
    ↓
验证成功，执行注册/重置密码
```

## 后续优化建议

1. **使用Redis存储验证码**
   - 当前：内存字典（服务器重启丢失）
   - 建议：Redis（持久化、支持过期自动清理）

2. **添加发送频率限制**
   - 防止短信轰炸
   - 使用IP或手机号限制

3. **验证码加密传输**
   - 防止中间人攻击
   - 使用HTTPS + 加密

4. **添加验证码日志**
   - 记录所有发送记录
   - 用于审计和问题追踪
