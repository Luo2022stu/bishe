# 失物招领认领流程功能说明

## 功能概述

本系统实现了完整的失物招领认领流程，包括认领请求、好友系统、拾取确认和智能柜取件码功能，有效避免恶意认领，提高安全性。

---

## 一、认领请求功能

### 1.1 发起认领

**前端交互**：
- 用户在物品详情页（`item-detail.html`）点击"💬 去认领"按钮
- 弹出输入框让用户填写认领留言
- 提交后发送认领请求给物品发布者

**后端API**：
```
POST /api/claims
```

**请求参数**：
```json
{
  "item_id": 物品ID,
  "message": "认领留言"
}
```

**返回结果**：
- 认领请求已发送给发布者
- 发布者收到系统通知
- 询问用户是否直接跳转到聊天页面联系发布者

---

### 1.2 接受/拒绝认领

**接受认领API**：
```
POST /api/claims/<claim_id>/accept
```

**拒绝认领API**：
```
POST /api/claims/<claim_id>/reject
```

**业务逻辑**：
- 只有物品发布者可以操作
- 接受认领后，自动拒绝该物品的其他待处理认领请求
- 更新物品状态为"已认领"（claimed）
- 发送通知给相关用户

---

## 二、好友系统功能

### 2.1 添加好友

**API**：
```
POST /api/friends
```

**功能**：
- 发送好友请求
- 接收方收到通知
- 状态：pending（待验证）、accepted（已通过）、rejected（已拒绝）

### 2.2 接受/拒绝好友请求

**接受好友API**：
```
POST /api/friends/<friendship_id>/accept
```

**拒绝好友API**：
```
POST /api/friends/<friendship_id>/reject
```

### 2.3 查看好友列表

**API**：
```
GET /api/friends
```

**返回**：
- 已接受的好友列表
- 包含用户基本信息

### 2.4 查看待处理好友请求

**API**：
```
GET /api/friends/pending
```

**返回**：
- 发送给当前用户的待处理好友请求

## 三、聊天消息好友限制

### 3.1 非好友限制规则

**查看聊天记录**（`GET /api/chat/messages/<user_id>`）：
- 如果不是好友且有历史消息 → 可以查看
- 如果不是好友且无历史消息 → 拒绝访问

**发送消息**（`POST /api/chat/messages/send`）：
- 如果不是好友 → 只能发送**一条**消息
- 第二条及后续消息 → 返回错误提示"需要添加对方为好友"
- 如果是好友 → 无限制发送

### 3.2 认领场景中的聊天

- 认领人发起认领后，可以直接与发布者聊天
- 发送第一条消息后，系统提示"需要添加好友才能继续发送"
- 双方通过认领详情中的通知可以添加好友
- 添加好友后，可无限制发送消息

---

## 四、拾取确认功能

### 4.1 确认拾取

**前端交互**：
- 用户在失物详情页看到"✅ 确认拾取"按钮（仅对失物类型显示）
- 点击后弹出确认对话框
- 确认后失主收到通知

**API**：
```
POST /api/items/<item_id>/pickup
```

**业务逻辑**：
- 只能确认拾取失物（lost类型）
- 不能拾取自己发布的物品
- 创建认领记录（状态为accepted）
- 发送通知给失主

**流程选项**：
1. 直接与失主联系（跳转到聊天）
2. 前往智能柜存放物品

---

## 五、智能柜取件码功能

### 5.1 将物品存入智能柜

**API**：
```
POST /api/claims/<claim_id>/locker-store
```

**请求参数**：
```json
{
  "locker_id": 智能柜ID
}
```

**业务流程**：
1. 验证认领人权限
2. 检查存储柜是否可用
3. 生成6位取件码
4. 创建存储物品记录
5. 更新认领请求状态（记录取件码）
6. 发送短信给失主（包含取件码和柜号）
7. 发送系统通知给失主

### 5.2 短信通知内容

```
【校园失物招领系统】您有一个物品待取：{物品名称}，柜号：{柜号}，取件码：{取件码}。请于7天内凭取件码取件。
```

### 5.3 系统通知内容

```
您认领的物品 {物品标题} 已由认领人存入智能柜 {柜号}，取件码：{取件码}
```

### 5.4 取件流程

失主收到取件码后：
1. 前往智能柜页面
2. 输入取件码和柜号
3. 系统验证并开柜
4. 物品状态更新为"已取件"

## 六、完整流程示例

### 场景1：拾到失物并发布招领

```
1. 用户A捡到物品 → 发布"失物招领"
2. 用户B在招领页面浏览 → 点击"去认领"
3. 用户B填写认领留言 → 提交认领请求
4. 用户A收到通知 → 查看认领请求
5. 用户A接受认领 → 物品状态变为"已认领"
6. 用户B收到通知 → 询问是否联系用户A
7. 用户B跳转聊天 → 发送第一条消息
8. 系统提示"需要添加好友" → 双方添加好友
9. 添加好友后 → 无限制聊天
10. 用户A将物品存入智能柜 → 生成取件码
11. 用户B收到短信和通知 → 凭取件码取物
```

### 场景2：丢失物品并寻找

```
1. 用户A发布"失物找寻"
2. 用户B捡到物品 → 在失物详情页点击"确认拾取"
3. 用户A收到通知 → 有用户确认拾取
4. 用户A选择联系用户B或等待用户B存柜
5. 用户B将物品存入智能柜 → 生成取件码
6. 用户A收到取件码短信 → 前往智能柜取物
```

## 七、安全机制

### 7.1 防止恶意认领

- 认领前必须注册登录
- 认领请求需要发布者确认
- 只能认领他人发布的物品
- 一个物品只能有一个认领成功
- 被拒绝的认领会收到通知

### 7.2 聊天保护

- 非好友只能发送一条消息
- 防止骚扰和恶意消息
- 需要双方同意才能无限制聊天

### 7.3 取件码安全

- 6位随机数字
- 7天有效期
- 一次性使用
- 短信+通知双重提醒

## 十、使用提示

### 发布者注意事项

1. 收到认领请求后，仔细核实物品描述
2. 可以要求认领人提供更多细节证明
3. 接受认领前可以先通过聊天沟通
4. 将物品存入智能柜时，确保物品完整

### 认领者注意事项

1. 认领时提供详细的物品描述
2. 准备好相关证明（照片、特征等）
3. 拾到物品后及时确认拾取
4. 存入智能柜前确认失主身份
5. 取件码请妥善保管，勿泄露给他人

### 常见问题

**Q: 认领后如何取物？**
A: 发布者接受认领后，可以选择直接联系取物或存入智能柜。存入智能柜后，您将收到取件码短信。

**Q: 为什么只能发一条消息？**
A: 这是防骚扰机制，添加对方为好友后可无限制发送消息。

**Q: 取件码有效期多久？**
A: 取件码有效期为7天，过期后需要重新申请。

**Q: 如何查看我的认领记录？**
A: 可以通过个人中心或调用 `/api/claims` API 查看。
