# 智能存储柜过期物品管理功能

## 功能概述

当物品在存储柜中存储超过7天后,系统会自动通知管理员进行清理。

## 实现的技术

### 1. 定时任务
- 使用Python的`threading`模块实现后台定时任务
- 每隔1小时自动检查一次所有待取状态的物品
- 检查物品的过期时间(`expires_at`)是否已过当前时间

### 2. 通知系统
- 基于现有的`UserNotification`模型
- 为每个管理员创建类型为`admin_alert`的通知
- 通知内容包括:
  - 物品名称
  - 存储柜号
  - 存入时间
  - 过期时间
  - 取件码
  - 清理提示

### 3. 去重机制
- 避免对同一物品重复通知
- 通过查询`UserNotification`表判断是否已发送过该物品的过期通知
- 每个过期物品只会通知管理员一次

### 4. 管理员API接口

#### 手动检查过期物品
```http
POST /api/admin/lockers/check-expired
Authorization: Bearer token_<user_id>
```

#### 获取所有过期物品
```http
GET /api/admin/lockers/expired-items
Authorization: Bearer token_<user_id>
```

返回所有状态为`pending`且已过期的物品列表。

#### 清理过期物品
```http
DELETE /api/admin/lockers/clear-expired/<item_id>
Authorization: Bearer token_<user_id>
```

清理操作会:
1. 将物品状态更新为`expired`
2. 记录清理时间
3. 释放对应的存储柜状态为`available`

## 工作流程

1. **物品存入**
   - 用户将物品存入存储柜
   - 系统生成6位取件码
   - 设置过期时间为当前时间+7天

2. **定时检查**
   - 后台线程每小时执行一次
   - 查询所有`status='pending'`且`expires_at <= 当前时间`的物品
   - 为每个过期物品发送通知给所有管理员

3. **管理员接收通知**
   - 管理员登录后可以看到通知
   - 通知详情包含完整的物品信息

4. **管理员清理物品**
   - 管理员查看过期物品列表
   - 选择清理操作
   - 系统更新物品状态并释放存储柜

## 核心代码

### 发送管理员通知函数
```python
def send_admin_notification(title, content, related_id=None, related_type='locker_item'):
    """发送通知给所有管理员"""
    admins = User.query.filter_by(role='admin').all()
    for admin in admins:
        notification = UserNotification(
            user_id=admin.id,
            type='admin_alert',
            title=title,
            content=content,
            related_id=related_id,
            related_type=related_type
        )
        db.session.add(notification)
    db.session.commit()
```

### 检查过期物品函数
```python
def check_expired_locker_items():
    """检查过期的存储物品并通知管理员"""
    current_time = get_beijing_time()
    expired_items = LockerItem.query.filter(
        LockerItem.status == 'pending',
        LockerItem.expires_at <= current_time
    ).all()
    
    for item in expired_items:
        # 检查是否已通知过
        existing_notification = UserNotification.query.filter_by(
            type='admin_alert',
            related_id=item.id,
            related_type='locker_item_expired'
        ).first()
        
        if not existing_notification:
            # 发送通知
            title = f'物品过期提醒 - {item.item_name}'
            content = f'物品 "{item.item_name}" 在存储柜中已超过7天未取走...'
            send_admin_notification(title, content, item.id, 'locker_item_expired')
```

### 启动定时任务
```python
def start_expired_item_checker():
    """启动定时任务,每隔1小时检查一次过期物品"""
    def check_loop():
        while True:
            try:
                check_expired_locker_items()
            except Exception as e:
                print(f'[智能柜] 定时任务异常: {e}')
            time.sleep(3600)  # 每小时检查一次
    
    checker_thread = threading.Thread(target=check_loop, daemon=True)
    checker_thread.start()
```

## 使用说明

1. 应用启动时自动启动定时任务
2. 物品存入7天后,系统会自动通知所有管理员
3. 管理员可以:
   - 查看过期物品列表
   - 手动触发检查
   - 清理过期物品
4. 清理后的物品状态变为`expired`,存储柜自动释放

## 注意事项

- 定时任务使用守护线程,应用退出时会自动停止
- 通知去重基于`related_id`和`related_type`的组合
- 每个过期物品只会通知一次,避免重复骚扰管理员
- 管理员清理后,存储柜状态自动变为可用
